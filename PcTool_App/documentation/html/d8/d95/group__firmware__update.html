#define ETX_OTA_DATA_MAX_SIZE   (1024U)

Designated maximum expected data size in an ETX OTA Packet.

Note
This definition's value does not stand for the size of the entire ETX OTA Packet. Instead, it represents the size of the "Data" field that is inside an ETX OTA Packet.

◆ ETX_OTA_Command

enum ETX_OTA_Command

ETX OTA Commands definitions.

These are the different Commands that the host can request to our MCU whenever the host sends an ETX OTA Command Type Packet to our MCU.

Enumerator
ETX_OTA_CMD_START 

OTA Application Firmware Start Command.

This command indicates to our MCU that the host wants to start a Firmware Update and that the following Data Type Packets that our MCU will receive are expected to contain a Firmware Image.

ETX_OTA_CMD_END 

OTA End command.

This command indicates to our MCU that the host wants to confirm with our MCU to having completely finished the ETX OTA Firmware Update process.

ETX_OTA_CMD_ABORT 

OTA Abort Command.

This command is used by the host to request to our MCU the instruction of aborting whatever ETX OTA Firmware Update process our MCU is at.

Note
Unlike the other Commands, this one can be legally requested to our MCU at any time and as many times as the host wants to.

Enumerator
ETX_OTA_PACKET_TYPE_CMD 

ETX OTA Command Type Packet.

This Packet Type is expected to be send by the host to our MCU to request a certain ETX OTA Command to our MCU (see ETX_OTA_Command ).

ETX_OTA_PACKET_TYPE_DATA 

ETX OTA Data Type Packet.

This Packet Type will contain either the full or a part/chunk of a Firmware Image being send from the host to our MCU.

ETX_OTA_PACKET_TYPE_HEADER 

ETX OTA Header Type Packet.

This Packet Type will provide the size in bytes of the Firmware Image that our MCU will receive, its recorded 32-bits CRC and the sub-type of the ETX OTA Data Type Packets to be received (i.e., ETX_OTA_Data_Packet_Type_t ).

ETX_OTA_PACKET_TYPE_RESPONSE 

ETX OTA Response Type Packet.

This Packet Type contains a response from our MCU that is given to the host to indicate to it whether or not our MCU was able to successfully process the latest request or Packet from the host.

Response Status definitions available in the ETX OTA Firmware Update process.

Whenever the host sends to our MCU a Packet, depending on whether our MCU was able to successfully process the data of that Packet of not, our MCU will respond back to the host with a Response Type Packet containing the value of a Response Status correspondingly.

◆ ETX_OTA_State

enum ETX_OTA_State

ETX OTA process states.

The ETX OTA process states are used in the functions of the Firmware_update module to either indicate or identify in what part of the whole ETX OTA process is our MCU currently at.

The ETX OTA process consists of several sub-processes or states that must be given place in the following orderly fashion:

  1. Idle State
  2. Start State
  3. Header State
  4. Data State
  5. End State
Note
If the ETX OTA process gives place in a different order as shown above (e.g., Idle State --> Start State --> Data State ..., where instead, the order of the states expected were Idle State --> Start State --> Header State ...), then the ETX OTA process will be terminated with the corresponding ETX_OTA_Status Exception code.
Enumerator
ETX_OTA_STATE_IDLE 

ETX OTA Idle State.

This state stands for when our MCU is not in a ETX OTA Firmware Update.

ETX_OTA_STATE_START 

ETX OTA Start State.

This state gives place when our MCU receives a Command Type Packet from the host right after leaving the ETX OTA Idle State. Our MCU will expect that Packet to contain the Start Command so that our MCU starts an ETX OTA Firmware Update.

ETX_OTA_STATE_HEADER 

ETX OTA Header State.

This state starts right after the Command Type Packet of the ETX OTA Start State is processed. In the ETX OTA Header State, our MCU will expect to receive a Header Type Packet in which our MCU will obtain the size in bytes of the Firmware Image to be received, its recorded 32-bits CRC and the sub-type of the ETX OTA Data Type Packets to be received (i.e., ETX_OTA_Data_Packet_Type_t ).

ETX_OTA_STATE_DATA 

ETX OTA Data State.

This state starts right after the Header Type Packet of the ETX OTA Header State is processed. In the ETX OTA Data State, our MCU will expect to receive one or more Data Type Packets from which our MCU will receive a Firmware Image from the host. It is also in this State where our MCU will apply that Firmware Image to itself.

ETX_OTA_STATE_END 

ETX OTA End State.

This state starts right after the Data Type Packet(s) of the ETX OTA Data State is/are processed. In the ETX OTA End State, our MCU will expect to receive a Command Type Packet containing the End Command so that our MCU confirms the conclusion of the ETX OTA Firmware Update.
