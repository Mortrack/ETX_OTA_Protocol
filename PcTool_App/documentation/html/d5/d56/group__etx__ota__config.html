Detailed Description

This file contains all the Default ETX OTA Protocol Configurations.

Note
It is highly suggested not to directly edit the Configurations Settings defined in this file. Instead of doing that whenever requiring different Configuration Settings, it is suggested to do that instead in an additional header file named as "app_etx_ota_config.h" whose File Path should be located as a sibbling of this Default ETX OTA Protocol Configuration File header file. However, to enable the use of that additional header file, you will also need to change the ENABLE_APP_ETX_OTA_CONFIG define, that is located at Default ETX OTA Protocol Configuration File, to a value of 1.
For more details on how to use this Default ETX OTA Protocol Configuration File , see the Readme.md file of the project on which the ETX OTA Protocol library is being used at.

◆ RS232_MODE_DATA_BITS

#define RS232_MODE_DATA_BITS   ('8')

Chosen Data-bits string with which we want the host to run the RS232 protocol.

The valid data-bits in the Teuniz RS232 Library are '5', '6', '7' and '8'.

◆ RS232_MODE_PARITY

#define RS232_MODE_PARITY   ('N')

Chosen Parity string with which we want the host to run the RS232 protocol.

The valid parity values in the Teuniz RS232 Library are:
- 'N' = None (i.e., No parity bit sent at all).
- 'O' = Odd (i.e., Set to mark if the number of 1s in data bits is odd.
- 'E' = Even (i.e., Set to space if the number of 1s in data bits is even.

◆ RS232_MODE_STOPBITS

#define RS232_MODE_STOPBITS   ('1')

Chosen Stop-bit string with which we want the host to run the RS232 protocol.

Note
The possible stopbit values in RS232 are '1' or '2'.

◆ TEUNIZ_LIB_POLL_COMPORT_DELAY

#define TEUNIZ_LIB_POLL_COMPORT_DELAY   (500000)

Designated delay in microseconds that it is to be requested to apply each time before calling the RS232_PollComport function.

For all the calls to the RS232_PollComport function, this definition is called once, except in the send_etx_ota_data function, where this definition is called twice.

Note
The Teuniz RS232 Library suggests to place an interval of 100 milliseconds, but it did not worked for me that way. Instead, it worked for me with 500ms.

◆ TRY_AGAIN_SENDING_FWI_DELAY

#define TRY_AGAIN_SENDING_FWI_DELAY   (9000000)

Designated delay in microseconds that it is to be requested to apply in case that starting an ETX OTA Transaction fails once only.

Note
The slave device sometimes does not get the start of an ETX OTA Transaction after its UART Timeout expires, which is expected since there is some code in the loop that the slave device has there that makes it do something else before waiting again for an ETX OTA Transaction, but that should be evaded by making a second attempt with the delay established in this variable.
