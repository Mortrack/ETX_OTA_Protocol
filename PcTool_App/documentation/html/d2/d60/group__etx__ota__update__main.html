◆ ETX_OTA_DATA_MAX_SIZE

#define ETX_OTA_DATA_MAX_SIZE   (1024U)

Designated maximum expected data size in an ETX OTA Packet.

Note
This definition's value does not stand for the size of the entire ETX OTA Packet. Instead, it represents the size of the "Data" field that is inside an ETX OTA Packet.

◆ Command_Line_Arguments

enum Command_Line_Arguments

Command Line Arguments definitions.

These definitions are used as identifiers for the Command Line Arguments that are received right after the resulting compiled file from this Main ETX OTA Update is executed via the terminal window.

Note
For more details see main .
Enumerator
TERMINAL_WINDOW_EXECUTION_COMMAND 

Command Line Argument Index 0, which should contain the string of the literal terminal window command used by the user to execute the Main ETX OTA Update program.

COMPORT_NUMBER 

Command Line Argument Index 1, which should contain the Comport with which the user wants the Main ETX OTA Update program to establish a connection with via RS232 protocol.

FIRMWARE_IMAGE_PATH 

Command Line Argument Index 2, which should contain the File Path, with respect to the File Location of the Main ETX OTA Update program, to the Firmware Image that the user wants this program to load and send towards a desired MCU for an ETX OTA Firmware Update.

ETX_OTA_PAYLOAD_TYPE 

Command Line Argument Index 3, which should contain the ETX OTA Payload Type to indicate to the Main ETX OTA Update program the type of Firmware Image that will be given.

◆ ETX_OTA_Command

enum ETX_OTA_Command

ETX OTA Commands definitions.

These are the different Commands that the host can request to our MCU whenever the host sends an ETX OTA Command Type Packet to our MCU.

Enumerator
ETX_OTA_CMD_START 

OTA Application Firmware Start Command.

This command indicates to our MCU that the host wants to start a Firmware Update and that the following Data Type Packets that our MCU will receive are expected to contain a Firmware Image.

ETX_OTA_CMD_END 

OTA End command.

This command indicates to our MCU that the host wants to confirm with our MCU to having completely finished the ETX OTA Firmware Update process.

ETX_OTA_CMD_ABORT 

OTA Abort Command.

This command is used by the host to request to our MCU the instruction of aborting whatever ETX OTA Firmware Update process our MCU is at.

Note
Unlike the other Commands, this one can be legally requested to our MCU at any time and as many times as the host wants to.

◆ ETX_OTA_Packet_t

enum ETX_OTA_Packet_t

Packet Type definitions available in the ETX OTA Firmware Update process.

Enumerator
ETX_OTA_PACKET_TYPE_CMD 

ETX OTA Command Type Packet.

This Packet Type is expected to be send by the host to our MCU to request a certain ETX OTA Command to our MCU (see ETX_OTA_Command ).

ETX_OTA_PACKET_TYPE_DATA 

ETX OTA Data Type Packet.

This Packet Type will contain either the full or a part/chunk of a Firmware Image being send from the host to our MCU.

ETX_OTA_PACKET_TYPE_HEADER 

ETX OTA Header Type Packet.

This Packet Type will provide the size in bytes of the Firmware Image that our MCU will receive, its recorded 32-bits CRC and the sub-type of the ETX OTA Data Type Packets to be received (i.e., ETX_OTA_Payload_t ).

ETX_OTA_PACKET_TYPE_RESPONSE 

ETX OTA Response Type Packet.

This Packet Type contains a response from our MCU that is given to the host to indicate to it whether or not our MCU was able to successfully process the latest request or Packet from the host.

◆ ETX_OTA_Payload_t

enum ETX_OTA_Payload_t

Payload Type definitions available in the ETX OTA Firmware Update process.

Whenever the host sends to our MCU a ETX_OTA_PACKET_TYPE_DATA Packet Type (i.e., a Data Type Packet), there are several possible sub-types for this and any other Data Type Packet that indicate the type of data that our MCU should expect to receive from the whole Data Type Packets that it will receive. These sub-types are given by ETX_OTA_Payload_t .

Enumerator
ETX_OTA_Application_Firmware_Image 

ETX OTA Application Firmware Image Data Packet Type.

ETX_OTA_Bootloader_Firmware_Image 

ETX OTA Bootloader Firmware Image Data Packet Type.

◆ ETX_OTA_Response_Status

enum ETX_OTA_Response_Status

Response Status definitions available in the ETX OTA Firmware Update process.

Whenever the host sends to our MCU a Packet, depending on whether our MCU was able to successfully process the data of that Packet of not, our MCU will respond back to the host with a Response Type Packet containing the value of a Response Status correspondingly.

Enumerator
ETX_OTA_ACK 

Acknowledge (ACK) data byte used in an ETX OTA Response Type Packet to indicate to the host that the latest ETX OTA Packet has been processed successfully by our MCU.

ETX_OTA_NACK 

Not Acknowledge (NACK) data byte used in an ETX OTA Response Type Packet to indicate to the host that the latest ETX OTA Packet has not been processed successfully by our MCU.

◆ ETX_OTA_Status

enum ETX_OTA_Status

ETX OTA Exception codes.

These Exception Codes are returned by the functions of the firmware_update module to indicate the resulting status of having executed the process contained in each of those functions. For example, to indicate that the process executed by a certain function was successful or that it has failed.

Enumerator
ETX_OTA_EC_OK 

ETX OTA Process was successful.

Note
The code from the HAL_ret_handler function contemplates that this value will match the one given for HAL_OK from HAL_StatusTypeDef .
ETX_OTA_EC_STOP 

ETX OTA Process has been stopped.

ETX_OTA_EC_NR 

ETX OTA Process has concluded with no response from Host.

ETX_OTA_EC_NA 

ETX OTA Payload received Not Applicable.

ETX_OTA_EC_ERR 

ETX OTA Process has failed.

Local variable checksum: Will hold the resulting checksum of the 32-bit CRC Hash Function to be calculated.

Note
A checksum is any sort of mathematical operation that it is performed on data to represent its number of bits in a transmission message. This is usually used by programmers to detect high-level errors within data transmissions. The way this is used is prior to transmission, such that every piece of data or file can be assigned a checksum value after running a cryptographic has function, which in this particular case, the has function is 32-bit CRC.

◆ main()

int main ( int  argc,
char *  argv[] 
)

Main part of the application program whose purpose is to receive some ETX OTA Payload Data from the user via Command Line Arguments to then send it to a user specified device by using the ETX OTA Protocol.

Parameters
argcContains the total number of Command Line Arguments that are given whenever executing the program contained in this file. The total number of arguments consider here will also include the command executed on the terminal window to run the program contained in this file. For example, if the Terminal Window Command used on the terminal window was
$./a.out 0 1 2
then
argc = 4
[in]argvHolds the actual Command Line Arguments that are given whenever executing the program contained in this file. Each argument will be allocated in a different index within the argv param, but were the first index will contain the command executed on the terminal window to run the program contained in this file and then the actual Command Line Arguments will be placed subsequently. For example, if the Terminal Window Command used on the terminal window was
$./a.out 0 1 2
then
argv[4] = {"./a.out", "0", "1", "2"}
Note
This main function expects to be given the following Command Line Arguments via the argv param:
  • Command Line Argument index 0 = TERMINAL_WINDOW_EXECUTION_COMMAND
  • Command Line Argument index 1 = COMPORT_NUMBER
  • Command Line Argument index 2 = FIRMWARE_IMAGE_PATH
  • Command Line Argument index 3 = ETX_OTA_DATA_PACKET_TYPE
